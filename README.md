How do you write programs that are maintainable, readable, and adaptable? Especially consider your work on the CRUD Python module from Project One, which you used to connect the dashboard widgets to the database in Project Two. What were the advantages of working in this way? How else could you use this CRUD Python module in the future?

In order to write maintable and readable programs, I try to follow all of the common programming rules for spacing and commenting. Commenting especially helps with readablity in order to describe what a certain block of code is there for. I tried to follow those rules through the Crud python module and project 2. This CRUD module would be very reusable with databases besides aac. It is designed to work with basically any other Mongo database, requiring only slight changes to make the switch.

How do you approach a problem as a computer scientist? Consider how you approached the database or dashboard requirements that Grazioso Salvare requested. How did your approach to this project differ from previous assignments in other courses? What techniques or strategies would you use in the future to create databases to meet other client requests?

Normally, when approaching a problem, I try to first take a step back and look at everything as a whole. I try to visualise what I want to happen, and how I want things to work. Then I take one step at a time. I do a little bit of work and then test it. Then I do a little bit more work and test that, and make sure it integrates well with the rest of what I've done. This process helps me to stay on track throughout big projects, such as this one, where a lot of the framework was already created. Most projects need to be started from scratch, so this one was a bit different since the framework was already created.

What do computer scientists do, and why does it matter? How would your work on this type of project help a company, like Grazioso Salvare, to do their work better?

Projects such as this one are all about making things easier for the client. The client (Grazioso Salvare) requested our help in this endeavor, because they wanted an easier time finding dogs to use for rescue training. It was our job to find ways to sort out the information in the austin animal shelters database that would help to find these certain dogs. So I would say that for the most part, the job of a computer scientist would be that of a problem solver. We are given all sorts of problems, and are tasked with finding the most user friendly solution to those problems!
